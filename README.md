# Grade Quiz

A mobile-first web app to search and quiz yourself on the parents and children in your child's grade.

Built with Next.js (App Router). No backend — the roster and photos are static files baked into the app, so it works identically on every device and deploys to Vercel in one click.

## How it works

- **Roster tab** — searchable list of every child and parent. Search matches names, jobs, companies, and related family members. Filter by All / Children / Parents.
- **Quiz tab** — pick one or more modes and a length, then answer multiple-choice questions with instant feedback and an end-of-round score + review.

### Quiz modes

| Mode | What it asks |
| --- | --- |
| Child → parent | Name a child's parent |
| Parent → child | Whose parent is this? |
| Child name from photo | Identify the child |
| Parent name from photo | Identify the parent |
| Child's parent from photo | Photo of child → name a parent |
| Parent's child from photo | Photo of parent → name their child |
| Parent's job | Job / company of a parent |

Photo modes only use people who have a photo; if too few do, the mode is disabled automatically.

## Your data

Everything lives in `public/`:

- **`public/roster.csv`** — one row per child, parents inline (up to 2). Shared parents are deduped by name, so siblings link to the same parent records.
- **`public/photos/`** — image files referenced by the `*_photo` columns.

### CSV columns

```
child_first_name, child_last_name, child_photo, child_grade, child_grade_color,
parent1_first_name, parent1_last_name, parent1_photo, parent1_job, parent1_company,
parent2_first_name, parent2_last_name, parent2_photo, parent2_job, parent2_company
```

- Names are split into first/last; the app joins them for display (multi-word surnames like `van der Gaast` are fine).
- `*_photo` is just a filename that exists in `public/photos/` (e.g. `octavia-dixon.jpg`). Leave blank for no photo — that person simply won't appear in photo-based questions.
- `child_grade` / `child_grade_color` (e.g. `G1`, `Blue`) are parsed and stored but not shown in the UI yet.
- A single-parent family just leaves the `parent2_*` columns blank.
- Any image format the browser supports works (`.jpg`, `.jpeg`, `.png`, `.webp`, …).
- Parents are deduped by full name, so siblings linked to the same parent share one record.

### Swapping in your real data

1. Replace `public/roster.csv` with your file (same columns).
2. Drop your images into `public/photos/`, named to match the `*_photo` cells.
3. Redeploy (or just refresh in dev).

The bundled sample data is generated by `npm run gen-sample` — handy if you want to regenerate placeholders, but you can ignore it once you have real data.

## Develop

```bash
npm install
npm run dev        # http://localhost:3000
```

## Deploy to Vercel

Push to a Git repo and import it at vercel.com, or:

```bash
npx vercel
```

No environment variables or configuration needed.
